Profile

I am seeking entry level, full time employment as either a general purpose programmer or web developer.

I program both single page, interactive, asynchronous web applications that have responsive design for phones, tablets, and computers, and also in general purpose programming languages. The technology that I use for web development is HTML, CSS, JavaScript, and SQL often with the frameworks of React and Express for the server and client and PostgreSQL for the database. Otherwise I use Java mostly, but also C++.

My first project, Web Text, provides the opportunity to store notes in folders once the account of a user is registered, authenticated, and authorized. It is hosted on Heroku. I wrote my second project, Minesweeper, in plain JavaScript. It is hosted on GitHub pages. Another project is a basic full stack implementation. It is written in plain JavaScript on the client side. The source code is on GitHub in two repositories, one for the client and another for the server and database, which were written with Express and PostgreSQL.

In addition to these projects, I have written in Java or JavaScript, translated from Java into JavaScript, or in a few instances adapted from JavaScript more than 150 algorithms. The data structures of the algorithms range from arrays and linked lists to stacks and graphs. I found prompts for them in different places, like Cracking the Coding Interview, LeetCode, HackerRank, freeCodeCamp, and InterviewBit. The algorithms are collected in a repository on my GitHub account. In another repository, there are my implementations of introductory programs of neural networks. I found the programs in various tutorials on YouTube. I wrote them in Python. Other projects on GitHub include an adaptation of a web crawler in JavaScript, a series of recreations of the front end of websites, and a series of grid games that are templates built from a template given by a freeCodeCamp video. A description of each and its link can be found on my resume. Of particular note among them is a grid game that uses Dijkstra's algorithm to find the shortest path in a grid of elements between an opponent and player as they move on a wraparound board. In August 2020, I wrote a book titled Algorithms and Programs. It is a rough draft under continual revision.

I have a 2017 MacBook Air that has the Mojave operating system. I use the Visual Studio Code text editor. My phone is an LG Tribute Dynasty.

I began to learn web development in 2018. At the time, I was a graduate student at The University of Texas at Austin in The Department of Anthropology. I had studied a number of languages at UT. They include Arabic, French, Hindi, Urdu, Chinese, Burmese, Latin, and Greek. As a child and teenager, I had studied Spanish, my first second language. Of the languages I studied at the university, I took the most courses in Hindi, and also studied it for four semesters at The American Institute of Indian Studies in Jaipur, Rajasthan. With world class instruction and a plethora of experiences in grammars and vocabularies from the Afro-Asiatic, Sino-Tibetan, and Indo-European family trees, I found myself at a crossroads in The Department of Anthropology with a methodological decision.

The decision was whether I should pursue the discipline of anthropology or another that is more focused on language. Because of my deep and abiding passion for morphology and syntax, in 2018, as an exploration into a potential new direction, I began to learn to program. I started with Python and immediately enjoyed it immensely. After working through a few textbooks, I changed languages, learning the syntax of JavaScript, HTML, CSS, and SQL. I started to program in Java and C++. With these experiences, I was convinced that I wanted to become a programmer. I left UT and enrolled in the remote course for web development at Thinkful Bloc. There I learned the technology for programming single page, interactive, asynchronous web applications with responsive design for multiple devices.

In late 2019, I visited Seattle and attended a few Meetups for web development. Having participated in and observed the community there, as well as having attended a few job fairs, I then returned to where I grew up, Austin, in early 2020 to look for employment. I attended a few job fairs through The University of Texas, which was kind enough to let me attend them for technology and science. It turns out that recruiters at job fairs encourage online application.

McDowell (CTC, 2014) calls online application a black hole from which one rarely receives an invitation to interview. Overall what I have found is that most companies want a candidate who has a technical degree or certificate. This is surprising since McDowell (CCI, 2008) says that the latter might be a stigma and hindrance during the application process.

Perhaps McDowell is incorrect, but I have it on good authority that her instruction is the North Star of preparatory material. Anyway, I learned to program with data structures and algorithms, and to do web development, which should make me a viable candidate.

I want to find an entry level, full time position. I would be happy to work remotely.

Otherwise, I am doing a lot of things. I am continuing the search for entry level, full time employment in spite of the anomalous and frightening pandemic. I am studying sorts, searches, and network theory. I am refining my programs, and writing new programs and algorithms. I am writing on Twitter about the scientific topics of climate change, evolution, and expansion. And I am renewing a study of Hindi and Spanish.
